AutoID — Auto Clock-In

Automated attendance tool for CMS. Opens Google Chrome via Playwright, loads a saved 30-day session, and clicks 線上打卡 at scheduled times with a humanised random delay.

Automatic sequence

  1. App starts → load schedule, options, lock state from JSON files and user.env
  2. Scheduler watches datetime.now() for a slot match
  3. Match found → compute random delay → start countdown
  4. Countdown = 0 → open Chrome in a daemon thread → click 線上打卡
  5. Browser stays open 3–10 min (random) → closes automatically
  6. Scheduler re-arms for the next slot

Algorithm

Random Delay — _calc_delay(opts)

# rand_enabled = True
delay = randint(min_min, max_min) × 60  +  randint(0, 59)   # seconds
# example: min=1, max=10  →  range 60 s … 659 s

# rand_enabled = False
delay = randint(0, 59)   # seconds  (⚠ can be 0)

Delay is computed at the moment the slot is detected, not in advance.

Next-Slot Scan — _next_fire_dt()

slots = [(t1h, t1m), (t2h, t2m)]

for days_ahead in 0 … 7:
    d = today + days_ahead
    if not is_day_on(d):  skip

    for (h, m) in sorted(slots):    # earlier time wins
        dt = datetime(d.year, d.month, d.day, h, m)
        if dt > now:  return dt

return None   # schedule OFF or no enabled day in window

is_day_on(d) — checks overrides[d.isoformat()] first, then defaults to weekday < 5.

Slot Detection — _tick_schedule()

purge _fired_keys older than today

if enabled AND today is ON:
    key = "YYYY-MM-DD HH:MM"
    if key NOT in _fired_keys:
        for (h, m) in slots:
            if now.hour == h AND now.minute == m:
                _fired_keys.add(key)
                _do_fire()
                break

re-arm:
    locked   → after(ms_until_next_:00)   # ~1 min  — minute boundary
    unlocked → after(30 000)               # every 30 seconds

_fired_keys is a set[str] — prevents double-trigger within the same slot minute regardless of how many times the poll fires that minute.

Minute-Alignment Formula

now = datetime.now()
ms_next = (60 - now.second) * 1000 - now.microsecond // 1000
after(max(ms_next, 500), callback)

Both _tick_clock and _tick_schedule (locked) use this formula, keeping the clock display and scheduler in sync at the true :00 boundary.

Potential Issues

#IssueNotes / Mitigation
1 Machine sleep / hibernate after() pauses while machine sleeps; resumes late. No missed-slot detection. Keep machine awake via Windows power settings.
2 Delay can be 0 s When random disabled, randint(0,59) can return 0 → browser fires at exactly :00. Fix: change to randint(1,59).
3 Session expiry A 1-hour pre-check runs before each slot. If the session is expired, an alert email is sent (if SMTP configured). Run Auth Test monthly or use Renew Auth.
4 No public holiday support Calendar defaults to weekday=ON. Toggle cells manually for national holidays.
5 Only 2 time slots Set both to same time if only 1 clock-in per day is needed.
6 2FA timeout too short Increase 2fa_timeout_secs in autoid_options.json if login is slow.
⚠ autoid_session.json contains browser cookies and localStorage. Keep it local — do not commit to version control.
