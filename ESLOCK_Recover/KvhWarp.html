🔐 KvhWarp 技術白皮書Technical Whitepaper

KvhWarp 是一個多層次檔案隱匿加密工具，專為速度與視覺匿蹤設計。

  • AES-256-GCM 認證加密（頭 / 尾）
  • AES-256-CTR 中段（Center Middle）與 B/C 步進 Range CTR
  • v2 格式：所有 magic / salt / metadata 均以密碼派生遮罩 XOR 混淆
  • 支援 Copy 模式（.ks 副本）與 In-place 模式（原地加密）
  • KDF：scrypt（建議）或 SHA-256（快速）

加密後的檔案以亂數時間戳命名（1745000000000000000.ks），原始檔名、大小與時間戳均加密存於 metadata 中。

KvhWarp is a multi-layer file stealth-encryption tool designed for speed and visual concealment.

  • AES-256-GCM authenticated encryption (head / tail)
  • AES-256-CTR for center middle and B/C-stride Range CTR
  • v2 format: all magic / salt / metadata XOR-masked with a password-derived mask
  • Supports Copy mode (.ks copy) and In-place mode (encrypt original)
  • KDF: scrypt (recommended) or SHA-256 (fast)

Encrypted files are renamed to random nanosecond timestamps (1745000000000000000.ks). Original filename, size and timestamps are stored encrypted inside the metadata.

🧱 加密層次架構Encryption Layer Architecture

每個加密層獨立作用於不重疊的位元組範圍（CTR 層可疊加） Each encryption layer operates on a non-overlapping byte range (CTR layers composable)

〰️ Range CTR 步進加密Stride Encryption

Range CTR 以 B/C 週期步進加密一段使用者自訂的位元組範圍：每 C bytes 中加密前 B bytes，跳過後 (C-B) bytes。CTR 計數器連續遞增（無論是否加密），確保可安全解密。

B/C 模式

模式說明B/C 計算
manual 直接指定 B（encrypt bytes）與 C（period bytes） 用戶設定，16 bytes 對齊
auto 指定加密比例 %，工具自動找最簡 B/C 分數 Fraction(pct, 100).limit_denominator(max_c)

Auto 模式範例：25% → B=1, C=4 → 每 64 bytes 加密前 16 bytes

重要：auto 模式在 warp_file 內部取得實際 range_size 後才解算 B/C，確保比例正確。

Range CTR encrypts a user-defined byte range using a B/C period stride: encrypt the first B bytes in each C-byte period, skip the remaining (C-B) bytes. The CTR counter increments continuously (encrypted or not) to enable correct decryption.

B/C Modes

ModeDescriptionB/C Calculation
manual Directly set B (encrypt bytes) and C (period bytes) User-specified, aligned to 16 bytes
auto Set a coverage %, tool finds simplest B/C fraction Fraction(pct, 100).limit_denominator(max_c)

Auto mode example: 25% → B=1, C=4 → encrypt first 16 bytes of every 64-byte period

Important: auto mode resolves B/C inside warp_file after the actual range_size is known, ensuring correct coverage.

選項預設說明

四道關卡（任一不過即略過壓縮）：

  1. 副檔名關卡：副檔名在 skip_exts 清單中 → 略過
  2. 大小關卡：檔案 > compress_max_mb → 略過
  3. 熵值關卡：Shannon 熵 > 6.8 bit/byte（高度壓縮格式）→ 略過；熵 < 5.5 使用 Zstd level 10，5.5–6.8 線性插值 level 3–9

Adaptive Compression (Copy and In-place Modes)

When enabled, the source file is compressed with Zstandard before encryption; decompression is applied automatically on unwarp. Works in both copy and in-place modes. Requires pip install zstandard — silently skipped if unavailable.

OptionDefaultDescription
compress_copyFalseEnable compression (GUI setting; applies to both copy and in-place modes)
compress_max_mb500Skip compression if file exceeds this MB (0 = unlimited)
compress_skip_extsbuilt-in listExtensions exempt from compression (already-compressed formats: mp4, zip, jpg…)

Four ordered gates (any failure silently skips compression):

  1. Extension gate: suffix in skip_exts list → skip
  2. Size gate: file > compress_max_mb → skip
  3. Entropy gate: Shannon entropy > 6.8 bit/byte (already dense) → skip; entropy < 5.5 uses Zstd level 10, 5.5–6.8 interpolated levels 3–9
  4. Savings guard: compressed size ≥ original × 0.95 → discard

🛡️ 安全分析Security Analysis

設計原則

  • 認證加密（AEAD）：GCM 層提供 Authentication Tag，防止靜默竄改
  • Oracle 防護：解密失敗路徑統一拋出相同 ValueError，防止旁路洩漏
  • 唯一 Nonce：每次加密均 os.urandom() 生成，防止重放攻擊
  • 鹽值隔離：scrypt 每個檔案獨立 16B 鹽值
  • Key 分離：Middle / Range CTR 使用 SHA256(key + suffix) 各自派生，不共用主 Key
  • Header 混淆：magic / salt / meta 均 XOR 密碼遮罩，阻擋 magic byte 掃描

威脅矩陣

威脅防護措施殘餘風險
暴力破解密碼scrypt (N=65536, r=8)低 弱密碼仍有風險
Magic byte 掃描識別XOR 密碼遮罩混淆 magic低
資料竄改GCM Authentication Tag低 僅 GCM 區塊有效
CTR 區塊竄改無 MAC（CTR 層無認證）中
元資料洩漏AES-GCM 加密 JSON metadata低
SHA-256 KDF 字典攻擊不建議使用 SHA-256 KDF高 弱密碼易破解
記憶體轉儲未清除密碼/密鑰記憶體中

Design Principles

  • Authenticated Encryption (AEAD): GCM provides Authentication Tag, preventing silent tampering
  • Oracle resistance: All decryption failure paths raise identical ValueError, no side-channel leakage
  • Unique nonces: os.urandom() per encryption, preventing replay attacks
  • Salt isolation: scrypt uses independent 16B salt per file
  • Key separation: Middle / Range CTR keys derived via SHA256(key + suffix), not sharing master key
  • Header obfuscation: magic / salt / meta all XOR-masked with password-derived mask, blocks magic byte scanning

⚠️ 限制與風險Limitations & Risks
KvhWarp 以隱匿為主要設計目標，不是軍事級完整檔案加密方案。
  • In-place 模式：中段明文保留在原始位置，大型影片中段內容未受保護
  • CTR 層（Middle / Range）不提供完整性認證，可被靜默修改
  • SHA-256 KDF 無抗暴力破解能力，不建議用於機密文件
  • 密碼未從記憶體清除（無 SecureString 機制）
  • 時間戳命名提供視覺匿蹤，但不防範已知日期/時間的關聯推斷
  • 不防範：記憶體傾印、鍵盤記錄器、實體存取、進階取證還原

建議使用情境

影片資料夾隱藏（平衡）In-place + scrypt + Middle CTR
重要文件（安全優先）Copy + scrypt + --all + tail
最高隱匿強度Copy + scrypt + tail + Middle + Range CTR 50%
KvhWarp is designed primarily for stealth — not military-grade full-file encryption.
  • In-place mode: middle plaintext remains at original position; large video middle content is unprotected
  • CTR layers (Middle / Range) provide no integrity authentication — can be silently modified
  • SHA-256 KDF has no brute-force resistance — not recommended for confidential documents
  • Password is not explicitly cleared from memory (no SecureString mechanism)
  • Timestamp naming provides visual stealth but doesn't prevent date/time correlation inference
  • Does not protect against: memory dumps, keyloggers, physical access, advanced forensic recovery
